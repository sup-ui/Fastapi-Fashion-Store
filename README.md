# FastAPI Fashion Store Backend

## Project Overview

This project is a backend API system for a Fashion Store built using **FastAPI**. The application simulates a real-world online fashion store where users can browse products, place orders, manage a wishlist, and explore items using search, sorting, and pagination features.

The project was developed as part of an internship training program to demonstrate practical backend development skills using Python and FastAPI. It focuses on designing RESTful APIs, handling data validation, implementing business logic, and building multi-step workflows.



## Features Implemented

This project includes several backend features that are commonly used in real-world applications:

### Product Management

* Retrieve all available products in the store
* Get product details by product ID
* View a summary of products including stock availability and category counts
* Add new products to the catalog
* Update product information such as price and stock status
* Delete products if there is no order history associated with them

### Order Management

* Create new orders with validated request data
* Calculate order totals using helper functions
* Apply seasonal discounts, bulk discounts, and optional gift wrap charges
* Retrieve all orders and calculate total revenue
* Search orders by customer name
* Sort orders based on total cost or quantity
* Paginate order results

### Wishlist Workflow

* Add products to a customer wishlist
* Prevent duplicate wishlist entries
* View all wishlist items and calculate the total wishlist value
* Remove items from the wishlist
* Convert all wishlist items into confirmed orders through a multi-step workflow

### Advanced Product Browsing

The application also provides advanced browsing capabilities:

* Keyword-based product search
* Product sorting by price, name, brand, or category
* Pagination for large product lists
* Combined browsing endpoint that supports filtering, searching, sorting, and pagination together


## Technologies Used

The backend system was developed using the following technologies:

* **Python** – Primary programming language
* **FastAPI** – High-performance framework for building APIs
* **Pydantic** – Data validation and request parsing
* **Uvicorn** – ASGI server used to run the FastAPI application
* **Swagger UI** – Automatic API documentation and testing interface



## API Testing

All endpoints were tested using **Swagger UI**, which is automatically generated by FastAPI.

After starting the server, the API documentation can be accessed at:

http://127.0.0.1:8000/docs

Swagger provides an interactive interface to test GET, POST, PUT, and DELETE endpoints.



## Project Structure

The project follows a simple and clear structure:

main.py – Contains all API endpoints, models, helper functions, and application logic
requirements.txt – Lists all dependencies required to run the project
README.md – Documentation explaining the project and its features
screenshots/ – Contains screenshots of tested API endpoints in Swagger UI



## How to Run the Project

Follow these steps to run the application locally:

1. Clone or download the repository.
2. Navigate to the project folder.
3. Install the required dependencies using the command:

pip install -r requirements.txt

4. Start the FastAPI server using:

uvicorn main:app --reload

5. Open your browser and go to:

http://127.0.0.1:8000/docs

You can now explore and test all the API endpoints using Swagger UI.
